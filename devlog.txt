Based on my understanding of VM, and the specs mentioned in the starter code:

    - We are working with a 32 bit address space (~4GB of virtual memory)

    - Physical RAM space is calculated by page size * frame count

    **Disclaimer** For now, frame count should only be a power of 2

    - A frame is a page in Physical RAM, indexed from 0 (maybe 1 if we want error code to be 0)

    - Assuming physical address space is always smaller than full virtual space
             => TODO: implement this

    - Physical RAM space may not be as big as entire virtual address space
         => Can't represent entire VA space in PA space
         => Need extra PA space to give illusion to processes that it has entire
           VA space
         => Swap space is allocated area in disk space that pages can live
           temporarily as OS deals with more active pages
         => allocateSwap() returns swap space id

    - uint32_t allocateFrame(void)
         => Acquires a new frame and returns the frame index
         => frame index ranges from 1 ~ (frame count * pageSize) - 1

    - uint32_t allocateSwap(void)
         => Allocates a new swap space and returns the swap space id (?)

    - copyToSwap(uint32_t frame, uint32_t swap)
         => Gets frame and "copies data" onto swap page
         => Increment page access count
         => Should raise error if invalid swap id detected

    - copyFromSwap(uint32_t swap, uint32_t frame)
         => Gets swap and "copies data" onto frame page
         => Increment page access count
         => Should raise error if invalid swap id detected

    - int initLibrary(struct config* conf)
         => I have no idea what this should be doing

    - void pageFault(uint32_t address)
         => Traces should be addresses

    - Do we implement TLB? or make students implement TLB?
         => Maybe with the limited global variables

    - Potential swappiness implementation

Necessary implementations:

    - Simulated RAM space based on page_size & frame_count
        => Set size
        => Array of page pointers

    - Simulated swap space that grows and has content
        => Only grows, never shrinks during trace
        => Also array of page pointers
            => Array as we need easy indexing

    - Traces that have virtual address load requests
        =>

    - Multi-level page tables could be implemented

    - Page table data structure:
        => Just an array of page table entries

    - Page table entry:
        => Necessary Fields:
            => Physical Page Number                    [< 32 - # of other bits]
            => Present/Absent (Valid) Bit              [1 bit]
        => Optional Fields to make it harder:
            => Referenced Bit:                         [1 bit]
                * 1 if being used by other processes
                * 0 otherwise
                * This could be for LRU purposes
            => Caching Bit:                            [1 bit]
                * 1 if should cache page
                * 0 if shouldn't cache
                * probably not needed cuz we want to
                  have all accesses from memory not
                  disk
            => Modified (Dirty) Bit:                   [1 bit]
                * 1 if should write back to disk
                * 0 if not
                * Next immediate possible bit to add
